Callbacks y promesas

Las callbacks y las promesas son dos conceptos clave en la programación asincrónica en JavaScript. Ambos se utilizan para manejar operaciones que toman tiempo, como las operaciones de red o las lecturas de archivos, de una manera que no bloquee el flujo del programa.

Una callback es una función que se pasa como argumento a otra función y se invoca una vez que se completa una operación asíncrona o algún evento ocurre. Las callbacks son una forma común de manejar operaciones asincrónicas en JavaScript, aunque pueden resultar en código anidado y difícil de mantener, lo que se conoce como "callback hell".






Las promesas son una forma más estructurada y legible de manejar operaciones asincrónicas. Una promesa representa un valor que puede estar disponible ahora, en el futuro o nunca. Las promesas tienen tres estados: pendiente, resuelta o rechazada. Se pueden encadenar para ejecutar código en respuesta al resultado de una operación asincrónica, lo que facilita el manejo de flujos de control complejos.
